## PART 1: UNIT ECONOMICS MODEL

### 1.1 Transaction Economics

The core business model is transaction-based, capturing a percentage of each credit trade on the platform.

#### Average Transaction Profile

| Metric | Value | Basis |
|--------|-------|-------|
| Average credits per transaction | 50 | Typical 10-dwelling development |
| Average credit price | £300/credit | Mid-market pricing |
| Average Transaction Value (ATV) | £15,000 | 50 × £300 |
| Platform fee rate | 4.0% | Competitive with EnTrade |
| **Gross Revenue per Transaction** | **£600** | £15,000 × 4% |

#### Variable Costs per Transaction

| Cost Item | Amount | % of Revenue | Notes |
|-----------|--------|--------------|-------|
| Payment processing | £30.20 | 5.0% | Stripe: 1.4% + 20p on £15K |
| Blockchain gas fees | £0.15 | 0.03% | Polygon L2 (~£0.05 × 3 txns) |
| Credit verification | £15.00 | 2.5% | Automated + spot checks |
| Customer support allocation | £12.00 | 2.0% | Tier 1 support per transaction |
| **Total Variable Costs** | **£57.35** | **9.6%** | |

#### Contribution Margin

| Metric | Value |
|--------|-------|
| Gross revenue per transaction | £600.00 |
| Variable costs per transaction | £57.35 |
| **Contribution margin** | **£542.65** |
| **Contribution margin %** | **90.4%** |
